# 管理者機能

## 👑 概要
管理者機能により、特定のユーザーが装置、予約、ユーザーなどのシステムデータを管理できるようになります。

## 🔑 アクセス制御
- **ロール**: 管理者 (`users` コレクション内の `isAdmin == true`)。
- **アクセス**: ホームページの「設定」アイコンから管理者メニューにアクセスできます。
- **セキュリティ**: Firestoreセキュリティルールにより、`equipments` および `locations` コレクションへの書き込み権限は管理者のみに制限されています。

## 🛠️ 機能

### 1. 装置管理
- **一覧表示**: ステータスインジケーター付きですべての装置を表示します。
- **追加/編集**: 新しい装置の作成や既存装置の変更を行います。
    - 項目: 名前、場所、説明、仕様、ステータス、画像URL。
- **削除**: 装置を削除します (実装に応じて論理削除または物理削除)。

### 2. 場所管理
- **追加/編集**: 装置が設置されている部屋/場所を管理します。

### 3. 予約管理
- **全体表示**: 全ユーザーのすべての予約を表示します。
- **強制キャンセル**: 管理者は任意の予約を削除できます。
- **編集**: 管理者は任意の予約を変更できます。

### 4. ユーザー管理 (予定)
- **ユーザー一覧**: 登録済みユーザーをすべて表示します。
- **管理者権限付与**: ユーザーを管理者に昇格させます。
- **ユーザー無効化**: アクセス権を取り消します。

## 📱 UIコンポーネント
- `AdminMenuPage`: 管理者アクション用ダッシュボード。
- `EquipmentManagementPage`: 装置のCRUDインターフェース。
- `EquipmentFormDialog`: 装置追加/編集用フォーム。
